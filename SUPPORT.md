# Support

Thanks for using Zarlania. Here is where to go for what.

## I have a question

Start a [discussion](https://github.com/Zarlania/zarlania-app/discussions). Questions
are not tracked as issues — issues represent committed work.

## Something is broken

Open a [bug report](https://github.com/Zarlania/zarlania-app/issues/new?template=bug_report.yml).
Please search existing issues first, and include the version and reproduction steps
— a bug that cannot be reproduced usually cannot be fixed.

## I want a new feature

Open a [feature request](https://github.com/Zarlania/zarlania-app/issues/new?template=feature_request.yml).
Describe the problem you are trying to solve rather than the solution you have in
mind; it often leads somewhere better.

## I found a security vulnerability

Do not open a public issue. Follow the [security policy](SECURITY.md).

## I want to contribute

Read [CONTRIBUTING.md](CONTRIBUTING.md). Note that every change requires a tracking
issue and a specific branch and pull-request naming convention, which CI enforces.

Issues labelled [`good first issue`](https://github.com/Zarlania/zarlania-app/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)
are scoped to be approachable starting points.

## Response times

This is an open-source project maintained in spare time. Issues and discussions are
reviewed as time allows, and there is no service-level agreement. Security reports
are prioritised — see [SECURITY.md](SECURITY.md) for those timelines.
